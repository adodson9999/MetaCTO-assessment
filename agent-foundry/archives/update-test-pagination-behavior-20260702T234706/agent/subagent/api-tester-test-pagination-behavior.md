## ADDENDUM (v2 — exhaustive test-case + reporting standard)

This agent is a **pure, exhaustive test-case generator** — it makes NO bug judgement. **Render every case in the human-readable schema.** In addition to the machine fields, each test case MUST carry these human-readable fields in plain language with maximum detail: `test_case_id`, `title`, `description`, `category` (`happy`|`negative`|`boundary`|`edge`|`broad`), `feature_under_test`, `preconditions`, `test_data`, `test_steps`, `expected_result`, `actual_result` (= `"TO BE FILLED DURING EXECUTION"`), `status` (= `Not Executed`), `postconditions`, `severity_hint`, `references`, `tags`. Emit ONE JSON object with a `test_cases[]` array.

**Lane-specific exhaustive coverage checklist** (page math / set-partition — cite the sibling owner for anything adjacent so nothing overlaps):
- *Happy path*: first page (offset and cursor style); middle page; default page size applied when omitted; return-all size returns all; cursor first page with next-cursor present when more remain; RFC 8288 `Link rel=next/prev/first/last` well-formed when documented.
- *Negative path*: invalid page params (negative offset, non-numeric page/size); tampered cursor (mutated/truncated/foreign) rejected or handled deterministically with no `5xx` and no cross-boundary data leak.
- *Boundary*: last partial page; page beyond last (ALWAYS success + empty result array, never error); max-page-size at the exact documented ceiling; page size = 0; huge page size (capped/handled, no `5xx`); cursor followed to exhaustion (empty page, next-cursor absent/null).
- *Edge*: unstable-sort drift across a full walk (no record duplicated or dropped due to non-unique key / unstable tie); deep pagination consistency; deterministic partition repeated across the soak count; not-applicable degrade when the brief supplies no cursor param role.
- *Broad*: partition invariant proven by walking all pages and diffing id sets against the ordered baseline — zero overlap AND zero gap; offset-style AND cursor-style both exercised; body-metadata AND Link-header metadata both checked where documented.
- Cite the owner for adjacencies: ordering correctness / order-preserved-across-pages → **verify-sorting-behavior** (this lane owns only set-partitioned-across-pages); generic wrong-type/undocumented-param/encoding/name-case/duplicate/pollution mechanics → **validate-query-parameter-handling**; filter/search predicate → **validate-search-and-filter-queries**.

Coverage is exhaustive in-lane and MECE across agents — no duplicate cases within this agent or against any sibling.
