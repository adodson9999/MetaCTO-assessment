### Lane-specific exhaustive coverage checklist (in-lane, MECE across agents)

- **Happy path**: create/update(PUT)/patch(partial UPDATE)/delete each emits exactly one entry in the window; read-audit records a read event; login and logout each audited; before/after values captured on update; CREATE entry actor `user_id` + `ip_address` + `timestamp` + `resource_id` all correct (audit-entry fidelity).
- **Negative path**: failed (wrong-credential) login audited as LOGIN_FAILED with attempted actor/source-ip; denied/unauthorized read audited as READ_DENIED with no resource data in the log body; a silent (missing) entry for any audited action is a coverage defect the judge flags.
- **Boundary**: timestamp exactly at the edge of `timestamp_tolerance_seconds`; ordered create→update→delete returns exactly three entries (no under/over-count) in monotonic non-decreasing timestamp order; window edge (entries just inside vs just outside the query window).
- **Edge (immutability / redaction / injection)**: an API attempt to MODIFY an existing entry rejected (403/405) with read-back proving the entry byte-for-byte unchanged; delete-of-audit-entry rejected; CRLF/newline/control-char/ANSI log-injection payload in a free-text field neutralized so it cannot forge or split a second entry; secret/PII field REDACTED so the cleartext value never appears anywhere in the entry.
- **Broad / combinatorial**: each audited action type (CREATE/READ/UPDATE/PATCH/DELETE/LOGIN/LOGOUT/LOGIN_FAILED/READ_DENIED) × exactly-one-entry coverage; each required field (user_id, action_type, resource_id, timestamp, ip_address) × fidelity assertion; each tamper vector (delete, modify) × rejection + unchanged read-back.

Adjacent concerns are cited to their owners: correlation-id echo / distributed-trace / span log fields → `validate-correlation-id-propagation` (existing handoff); generic response-schema → `validate-json-schema-responses`; generic error wording → `verify-error-message-clarity`; CRUD read-back lifecycle for its own sake → `verify-crud-operation-integrity` (here read-back exists only to PROVE audit-entry fidelity).

Coverage is exhaustive in-lane and MECE across agents — no duplicate cases within this agent or with any sibling.
