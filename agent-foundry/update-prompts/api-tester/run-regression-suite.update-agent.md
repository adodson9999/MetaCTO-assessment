## Change prompt (verbatim, exhaustive)

CONVERT this agent from a plan-only two-artifact comparator into a pure bug-finder that emits a `cases[]` matrix surfacing DEFECTS in a regression-report COMPARISON endpoint/artifact. Today it ingests two automated-test result artifacts (JUnit XML, Jest --json, pytest-json, TAP, TRX/NUnit) and COMPUTES total_tests, previously_passing, regressions, newly_passing, flaky, slowed, and overall_status, emitting those as a report. That is a calculator, not a bug-finder, and it must be reframed exactly like its api-tester siblings: it must now TEST the API surface that PARSES + COMPARES those regression reports and find bugs in it, by emitting cases whose `expected_by_contract` (the contract-correct parse/compare result and standard HTTP/REST behavior) is compared by a deterministic harness against the endpoint's `observed` output, every divergence surfaced as a deviation.

REMOVE the calculator behavior: this agent NO LONGER emits `total_tests`, `previously_passing`, `regressions`, `newly_passing`, `flaky`, `slowed`, or `overall_status` as computed outputs, and NO LONGER parses/compares the artifacts itself. Instead it emits the standard api-tester envelope: exactly ONE JSON object with top-level `agent`, `lane`, `baseline`, `cases` (array), and `out_of_scope`, and nothing else — no prose, no code fence. The parse/compare logic survives ONLY as the `expected_by_contract` value inside each case (the reference/oracle answer the comparator MUST return for a fixed pair of input artifacts); the agent asserts, it does not report.

NEW LANE: parse/compare defects in a regression-report comparator endpoint/artifact — the endpoint (referenced only by role, e.g. `regression_compare_endpoint`) that, given a previous-build (N-1) and current-build (N) test-result artifact (plus repeated N runs for flakiness) in any supported format, returns the comparison report. The agent surfaces format-parsing defects, miscounts, wrong status derivation, and malformed-artifact/standard-API-contract violations on that endpoint.

Every case mirrors the house case schema used by sibling api-testers: `label`, `role`, `endpoint_role`, `method`, `recipe` (an object with `kind` from the CLOSED vocabulary plus `artifact_format` and the input `fixture_role`(s) naming the N-1/N artifacts by role, and where relevant a `field_under_test`), `expected_by_contract` (the contract-correct comparison value or status + invariants, read from the oracle and the researched format specs — NEVER from the target's docs or observed behavior), `expected_by_docs` (only when docs differ), `also_accept`, and an `assertion` read-back block naming the exact response field(s) to compare (e.g. `{ "read_field": "overall_status", "compare": "equals", "value_source": "expected_by_contract" }`). Stay feature-agnostic and role-only: never name a real URL/path/host/test id; echo runtime-provided build ids and field names byte-for-byte. Keep the self-awareness / code-review clause (≥85, loop until it does) and the fail-closed out-of-lane sentinel naming the owning sibling in `out_of_scope`.

## Research basis
- JUnit XML schema: `<testsuites>`/`<testsuite>`/`<testcase>`; a testcase PASSED iff it has no `skipped`/`failure`/`error` child; `<failure>`/`<error>` carry message + type; anchors the parse and pass/fail rules (testmoapp/junitxml; windyroad JUnit.xsd).
- TAP: `TAP version 13`, plan line `1..N`, `ok`/`not ok N - desc`, `# SKIP`/`# TODO` directives — anchors the TAP parse and schema-version-mismatch case (testanything.org; pytest-tap).
- pytest-json / Jest --json / TRX-NUnit outcome fields anchor the cross-format equivalence and per-format parse cases.
- Contract-oracle rows: Validation (4xx not 5xx on malformed/empty/truncated artifact), AuthN (401), Status semantics, Documented-capability (schema conformance) anchor Groups D and E.
- Sources: testmoapp/junitxml (JUnit XML format spec); windyroad/JUnit-Schema JUnit.xsd; testanything.org TAP producers; pytest-json-report; PyPI pytest-tap.

## ADDENDUM (v2 — exhaustive test-case + reporting standard)

When running update-agent for this agent, pass the Change prompt above AND this ADDENDUM together as a single change prompt.

This agent is a pure, exhaustive test-case generator in its lane. It fills the Expected Result (the contract-correct parse/compare answer the regression-comparator endpoint MUST return for a fixed artifact pair, sourced from the contract oracle and the researched format specs) and leaves `actual_result = "TO BE FILLED DURING EXECUTION"` and `status = Not Executed`. It emits NO deviations, NO verdict, NO pass/fail — a separate judge agent submits each artifact pair, records the endpoint's real output, and decides whether it is a bug. This ADDENDUM is governed by `00-AUTHORING-STANDARD-exhaustive-testcases.md`.

Test Case ID prefix: TC-REGRESS-NNN (zero-padded, sequential, stable).

Render every case in the human-readable schema with these fields: `test_case_id`, `title`, `description`, `category` (happy|negative|boundary|edge|broad), `feature_under_test` (the regression-comparison endpoint that SERVES the metric), `preconditions`, `test_data`, `test_steps`, `expected_result`, `actual_result` (="TO BE FILLED DURING EXECUTION"), `status` (=`Not Executed`), `postconditions`, `severity_hint`, `references`, `tags`. Preserve this agent's existing machine fields (`label`, `role`, `endpoint_role`, `method`, `recipe` incl. `artifact_format`, `expected_by_contract`, `expected_by_docs`, `also_accept`, `assertion`) under a `machine` key inside each case. Emit ONE JSON object with a `test_cases[]` array.

### Lane-specific exhaustive coverage checklist (feature under test = the endpoint that parses/compares regression reports and serves the metric; in-lane, MECE across agents)

- **Happy path (correct computation returned)**: each supported format (JUnit XML, Jest --json, pytest-json, TAP, TRX/NUnit) parsed to the same logical comparison; cross-format equivalence (N-1 one format, N another) yields the format-independent result; total_tests = distinct testcase count; previously_passing correct; regressions/newly_passing/flaky/slowed counts correct; overall_status "fail" on regression, "pass" with no regression; schema-conformant 200 with all seven fields typed.
- **Negative path (malformed input → 4xx not 5xx)**: empty/zero-byte artifact → 4xx; malformed/unclosed-tag XML → 4xx not 5xx (no parser crash); truncated-mid-element artifact → 4xx; only one of two required artifacts → 4xx; missing/invalid credential → 401; unsupported schema/format version (`TAP version 99`) → 4xx not a silent misparse.
- **Boundary (zero-sample / threshold edges)**: a test whose N runtime exceeds N-1 × exactly the documented slowed multiple; a regression pair where exactly one test flips pass→fail; overall_status derivation at the no-regression-but-other-changes edge.
- **Edge (duplicate / outlier / encoding)**: duplicate fully-qualified testcase name → counted once (DISTINCT); a flaky test (both passes and fails across repeated N runs) placed in `flaky` and EXCLUDED from `regressions`; skips/errors NOT counted as passes; non-ASCII/UTF-8-BOM testcase names + non-C locale preserved byte-for-byte with counts unaffected.
- **Broad / combinatorial (each formula field × input shape)**: each report field (total_tests, previously_passing, regressions, newly_passing, flaky, slowed, overall_status) × a fixture isolating its correct value; each format × the same logical regression; each degenerate artifact (empty, malformed, truncated, version-mismatch, single) × 4xx-not-5xx class.

Adjacent concerns are cited to their owners: deep response-body JSON-schema strictness → `validate-json-schema-responses` (here schema case proves only the seven fields exist and are typed); error-message wording/leak → `verify-error-message-clarity` (here 4xx cases assert only status class + machine-readable error presence); credential lifecycle → `test-authentication-flows`; defect-density metric → `track-defect-density`; satisfaction metrics → `measure-api-consumer-satisfaction`.

Coverage is exhaustive in-lane and MECE across agents — no duplicate cases within this agent or with any sibling.
