## Research basis

- Kubernetes Gateway API / Envoy Gateway / Istio HTTPRoute docs (2026): path-prefix rewrite/strip, host-based and header-based match routing, weighted traffic split (e.g. 90/10 stable/canary), and default dot-segment/trailing-slash normalization enabled by default — grounds `host-header-based-routing`, `header-match-routing`, `canary-traffic-split-per-weight`, `trailing-slash-and-normalization-routing`.
- Ingress-NGINX "Before You Migrate" + GCP URL-rewrite guidance: prefix-rewrite to `/` can produce `//path` double-slash and normalization must not route a `/../` form outside the route map — grounds the path-traversal guard in the normalization case.
- RFC 9110 §15.5.15 (414 URI Too Long) + MDN/Zuplo/golang-go#79162 (431 Request Header Fields Too Large): the gateway/proxy is where oversized request-line and header fields are rejected, and multi-hop proxies that append X-Forwarded-For can grow headers into a 431 — grounds `oversized-uri-414-at-gateway`, `oversized-headers-431-at-gateway`, and the "did not append-and-forward oversized forwarding headers" assertion.
- RFC 9110 §15.5.6 (405 Method Not Allowed) requires an Allow header enumerating permitted methods — grounds `method-not-allowed-allow-header`.

## Gap summary

Original 7 cases covered single-backend delivery, one path-rewrite, unknown-route 404, method-not-allowed 405, load-balancing, injected headers, and service-down 503. Gaps closed: (1) route SELECTION was tested only by path — host-based and header-based selection (and their default fallbacks) were untested; (2) canary/weighted traffic SPLIT distinct from round-robin load-balancing was absent; (3) request-ENVELOPE-size rejection at the gateway (414 URI, 431 headers) was untested, including the multi-hop header-growth defect; (4) path NORMALIZATION routing (trailing slash, dot-segment, and traversal-outside-route-map) was untested; (5) the 405 lacked the mandatory Allow-header assertion. Every new backend-asserting case carries `other_backends` and asserts the request reached exactly one backend unchanged.
