## Research basis
- OWASP WSTG NoSQL Injection & PortSwigger NoSQL injection: operator injection via nested objects (`{"$ne":"x"}`, `$gt`, `$regex`, `$where`); sanitizers strip `$`/`.` keys — motivates cases 10-11 (sole-owner injection lane).
- Moesif / Speakeasy / Strapi REST filtering guides: multiple params = logical AND, comma-list = OR/union, operators `co/sw/ew/gte/lte`, EMPTY/NOT-EMPTY — motivates cases 1,2,3,4,5.
- Dreamfactory / oneuptime filtering-and-sorting: gt/lt/gte/lte numeric + date-range predicates, reserved-char encoding, case-insensitive matching — motivates cases 4,5,7,8,9.
- Contract-oracle "List + filter" row: every returned record matches predicate; count consistent; unknown filter per policy — the assertion backbone for all 16.

## ADDENDUM (v2 — exhaustive test-case + reporting standard)

When running update-agent for this agent, pass the Change prompt above AND this ADDENDUM together as a single change prompt.

This agent is a **pure, exhaustive test-case generator** — it makes NO bug judgement. It authors each case and fills the *Expected Result* (the definition of correct search/filter behavior, sourced from the contract oracle and the given spec). It leaves `actual_result` = `"TO BE FILLED DURING EXECUTION"` and sets `status` = `Not Executed`. It emits **no** deviations, verdicts, `is_bug` flags, or pass/fail counts — a **separate judging agent** executes the cases and decides whether an Actual Result is a bug. This addendum is governed by `00-AUTHORING-STANDARD-exhaustive-testcases.md`.

**Test Case ID prefix: `TC-SEARCH-NNN`** (zero-padded, sequential, stable across runs).

**Render every case in the human-readable schema.** In addition to the machine fields, each test case MUST carry these human-readable fields in plain language with maximum detail: `test_case_id`, `title`, `description`, `category` (`happy`|`negative`|`boundary`|`edge`|`broad`), `feature_under_test`, `preconditions`, `test_data`, `test_steps`, `expected_result`, `actual_result` (= `"TO BE FILLED DURING EXECUTION"`), `status` (= `Not Executed`), `postconditions`, `severity_hint`, `references`, `tags`. Preserve every existing machine field (`role`, `endpoint_role`, `method`, `recipe{kind,input_role,...}`, `assertion`, `also_accept`, `expected_by_contract`) as sub-fields under a `machine` key so the harness/judge still gets structured inputs while humans get the readable case. Emit ONE JSON object with a `test_cases[]` array.

**Lane-specific exhaustive coverage checklist** (predicate correctness + injection — cite the sibling owner for anything adjacent so nothing overlaps):
- *Happy path*: keyword/substring search returns exactly the matching set; single-category filter; multi-value OR/union filter; two-key AND composition; inclusive numeric range; inclusive date range; case- and accent-folded match per documented policy; every returned record re-verified against the predicate.
- *Negative path*: unknown-filter-VALUE rejected or empty per policy (never silently unfiltered); malformed range bounds; a filter value outside the documented domain returns error-or-empty, never all-records.
- *Boundary*: range bounds exactly at [lower, upper] with boundary records included; range where lower==upper; empty-result filter (valid value matching zero records → empty set, success class, never error/all); single-record match; whole-collection match (every record matches).
- *Edge*: wildcard/metacharacter (`%`/`_`/`*`) treated as literal-or-wildcard per policy with no over-broad match; unicode/homoglyph/accent-folded search terms; whitespace-only term; combined filter that intersects to empty.
- *Broad / injection (SOLE OWNER)*: NoSQL operator-object payload (`{"$ne":null}`/`$gt`/`$regex`/`$where`) treated as an opaque literal — set not broadened, never `5xx`; SQL boolean/tautology fragment (`' OR '1'='1`) does not bypass the predicate (returned set ≠ full table), never `5xx`; each documented filter key × valid/invalid value.
- Cite the owner for adjacencies: ordering correctness → **verify-sorting-behavior**; page math (first/middle/last/overlap/gap) → **test-pagination-behavior**; generic param mechanics (missing-required, wrong-type, undocumented-*parameter*, encoding, name-case, duplicate-key, pollution) → **validate-query-parameter-handling**; request-body constraints → **validate-request-payloads**. This lane owns predicate correctness and filter-operator injection only.

Coverage is exhaustive in-lane and MECE across agents — no duplicate cases within this agent or against any sibling.
