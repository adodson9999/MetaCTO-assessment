## Research basis
- RFC 9110 §7.6.1 (Connection & hop-by-hop): Connection, Keep-Alive, TE, Trailer, Transfer-Encoding, Upgrade, Proxy-Authorization, Proxy-Authenticate are connection-scoped and MUST NOT be forwarded by an intermediary; any field named inside `Connection:` is also consumed at the hop. Source: rfc-editor.org/rfc/rfc9110.html; MDN Connection header; nathandavison.com abusing-http-hop-by-hop-request-headers.
- HTTP request smuggling (TE.TE / CL.TE / duplicate CL / obfuscated TE): a request with both Transfer-Encoding and Content-Length, duplicate framing fields, obsolete line folding, or nonstandard whitespace is ambiguous and must be rejected/normalized, not forwarded as-is. Sources: portswigger.net/web-security/request-smuggling; owasp.org WSTG Testing for HTTP Request Smuggling; imperva.com http-request-smuggling.
- RFC 9110 §5.1 (field names case-insensitive), §5.2 (repeated fields fold by ordered comma-join), §5.4 (field limits / 431). W3C Trace Context: continue an inbound traceparent (same trace-id) + preserve tracestate; originate a fresh valid traceparent when none is present.
- contract-oracle.md "Headers" row (standard headers present & correct) and "Validation" row (malformed request never 5xx).
