## Research basis
- RFC 7578 (Returning Values from Forms: multipart/form-data): §4.1 boundary delimiter and required CRLF framing (Group A); §4.3 multiple files under one name; §4.4 per-part `Content-Type` defaulting to `text/plain`; §4.5 charset parameter per text part; and the rule that only Content-Disposition / Content-Type / limited Content-Transfer-Encoding headers are permitted and other MIME headers MUST be ignored (Group B).
- Multipart-parser production-failure guidance: most failures come from malformed boundaries, incorrect/omitted per-part headers, and unexpected payload shapes rather than the happy path (Groups A, B).
- Parser resource-bounding practice: capping per-request part count to prevent unbounded processing (Group C).
