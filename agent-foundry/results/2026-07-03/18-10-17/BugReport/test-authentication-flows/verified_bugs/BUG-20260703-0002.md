# BUG-20260703-0002 — GET /auth/me with a malformed/invalid token is rejected (401/403)

## Description
When I exercise the 'GET /auth/me with a malformed/invalid token is rejected (401/403)' check against GET /auth/me on http://localhost:8899, I expect status in [401, 403]; body has 'message'. but I get: HTTP 500. message='invalid signature' FAILED: status in [401, 403].

## Steps to Reproduce
1. Send GET /auth/me with a malformed Bearer token.
2. Assert status in [401, 403].
3. Assert body has 'message'.

## Actual vs. Expected
- **Expected:** status in [401, 403]; body has 'message'. (Per the documentation (data/documentation-reviewer/reference/auth.md:9 — https://dummyjson.com/docs/auth): "When the Authorization header is missing or the token is invalid, the API returns 401 Unauthorized.")
- **Actual:** HTTP 500. message='invalid signature' FAILED: status in [401, 403].

## Severity & Priority
- **Severity (technical impact):** CRITICAL — Blocker — crash, data loss, or auth/data exposure
- **Priority (business urgency):** P1 — High — fix now

- Target: http://localhost:8899
- API: DummyJSON clone (local)
- Runtime: Node.js
- Endpoint: GET /auth/me
- Run: RUN-20260703-181017 (2026-07-03 18:10:17)
