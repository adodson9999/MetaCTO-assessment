ADD the following NEW probe groups (each a new key in `matrix`, each a new probe `role` added to the closed off-enum probe vocabulary; every invalid probe `expected_class`:`4xx`, `also_accept`:`[]`; body = valid body with the one enum field set to the probe value):
- `case_variant_upper`: per enum field whose VALID_ENUMS are all lowercase-only, one body with that field's first value UPPERCASED, expected rejected (mirror of the existing lowercase `case_variant` for the opposite casing).
- `case_variant_mixed`: per enum field, one body with the first value's casing inverted per-character (e.g. tOGGLED), expected rejected — catches loose `equalsIgnoreCase`-style acceptance.
- `whitespace_internal`: per enum field, one body whose value is a valid enum value with an internal space inserted (not just padded), expected rejected.
- `whitespace_leading` / `whitespace_trailing`: per enum field, two bodies — one leading-space-only and one trailing-space-only variant — expected rejected (splits the existing both-sides `whitespace_padded` to isolate one-sided trim bugs).
- `trailing_chars`: per enum field, one body whose value is a valid enum value with extra trailing characters appended (e.g. `VALUE_1x`), expected rejected — catches prefix/`startsWith` matching bugs.
- `leading_zero`: per enum field whose VALID_ENUMS are numeric-looking or code-like, one body whose value is a valid value with a leading zero prepended (e.g. `01` for `1`), expected rejected — catches numeric-normalization membership bugs.
- `numeric_for_string_enum`: per string-typed enum field, one body whose value is a bare JSON number equal-in-text to a valid value (distinct from the existing `numeric_enum` out-of-set/stringified probes), expected rejected — isolates int-for-string-enum coercion.
- `boolean_for_enum`: per enum field, one body whose value is a JSON boolean, expected rejected (wrong-type refinement within the enum lane).
- `array_for_single`: per SINGLE-select enum field, one body whose value is a one-element array wrapping a valid value, expected rejected — catches multi-select-where-single-expected acceptance (the inverse of the existing `array_multi_select` group).
- `single_for_multi`: for the multi-select field, one body whose value is a bare scalar valid member (not wrapped in an array), expected rejected-or-per-policy — catches single-where-multi-expected coercion.
- `multi_select_empty_array` and `multi_select_duplicate`: for the multi-select field, one body with an empty selection array and one with a duplicated valid member — each expected per the multi-select contract (empty → per required/optional policy; duplicate → rejected or deduped, NEVER 5xx). NOTE: an empty selection array that is a PURE empty-state overlaps api-tester-validate-null-empty-fields ownership — emit `multi_select_empty_array` ONLY as an enum-MEMBERSHIP probe (does the empty set satisfy the enum-membership contract) and DEFER the pure empty-array STATE grading to the null agent, citing it.
- `unicode_look_alike_greek` / `unicode_look_alike_fullwidth`: per enum field, additional confusable-swap bodies beyond the existing Cyrillic look-alike — one Greek-letter homoglyph and one fullwidth-form variant of a valid value — each expected rejected, to broaden homoglyph/normalization coverage.
- `unicode_combining_mark`: per enum field, one body whose value is a valid value with a zero-width or combining diacritical mark inserted, expected rejected — catches normalize-then-compare gaps.
