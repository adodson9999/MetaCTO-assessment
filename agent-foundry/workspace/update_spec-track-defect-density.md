# Update Spec — track-defect-density

## User prompt
CONVERT this agent from a plan-only metric calculator into a pure bug-finder that emits a `cases[]` matrix surfacing DEFECTS in a defect-density REPORTING endpoint/artifact. Today it ingests a sprint brief and COMPUTES defect_density, severity_weighted_density, per_area_densities, rolling_3_sprint_average, deviation_percent, alert, p1–p4 counts, and trend, emitting those numbers as a report. That is a calculator, not a bug-finder, and it must be reframed exactly like its api-tester siblings: it must now TEST the API surface that PRODUCES/SERVES that metric and find bugs in it, by emitting cases whose `expected_by_contract` (the contract-correct computation and standard HTTP/REST behavior) is compared by a deterministic harness against the endpoint's `observed` output, every divergence surfaced as a deviation.

REMOVE the calculator behavior: this agent NO LONGER emits `sprint_name`, `defect_density`, `severity_weighted_density`, `per_area_densities`, `rolling_3_sprint_average`, `deviation_percent`, `alert`, `p1_count`..`p4_count`, or `trend` as computed outputs, and NO LONGER computes the metric itself. Instead it emits the standard api-tester envelope: exactly ONE JSON object with top-level `agent`, `lane`, `baseline`, `cases` (array), and `out_of_scope`, and nothing else — no prose, no code fence. The metric math survives ONLY as the `expected_by_contract` value inside each case (the reference/oracle answer the reporting endpoint MUST return for a fixed input fixture); the agent asserts, it does not report.

NEW LANE: contract/computation defects in a defect-density reporting endpoint/artifact — the endpoint (referenced only by role, e.g. `defect_density_report_endpoint`) that, given a sprint fixture (defect issues with P1–P4 priorities, a numstat diff, three preceding densities, a component-label map), returns the density report. The agent surfaces wrong math, wrong exclusions, wrong rollups, wrong windows, wrong thresholds, and standard API-contract violations on that endpoint.

Every case mirrors the house case schema used by sibling api-testers: `label`, `role`, `endpoint_role`, `method`, `recipe` (an object with `kind` from the CLOSED vocabulary plus a `fixture_role` naming the input fixture by role and, where relevant, a `field_under_test`), `expected_by_contract` (the contract-correct computed value or status + invariants, read from the oracle and the researched formulas — NEVER from the target's docs or observed behavior), `expected_by_docs` (only when docs differ), `also_accept`, and an `assertion` read-back block naming the exact response field(s) to compare (e.g. `{ "read_field": "defect_density", "compare": "equals", "value_source": "expected_by_contract" }`). Emit RECIPES only — never call the endpoint, never state a concrete numeric `observed`; the deterministic harness POSTs/GETs each fixture to the reporting endpoint and records the real response. Stay feature-agnostic and role-only: never name a real URL/path/host/sprint id/file/component; echo runtime-provided field names byte-for-byte. Keep the self-awareness / code-review clause (≥85, loop until it does) and the fail-closed out-of-lane sentinel naming the owning sibling in `out_of_scope`.

The canonical count is TWENTY (20) cases; a case is omitted only when its required fixture input (e.g. a per-area component map, a zero-changed-lines fixture) is not supplied — in which case fail the count check rather than fabricating a surface. Enumerate exactly:

### GROUP A — core density math (defects / non-test changed lines × 1000, half-up 2dp)
- label `density_correct_value`, role `density_math`, endpoint_role `defect_density_report_endpoint`, method POST, recipe `{ "kind": "density_computation", "fixture_role": "sprint_fixture", "field_under_test": "defect_density" }` — a known fixture whose contract-correct density = defects / (Σ added+deleted over NON-test files) × 1000 rounded half-up to 2dp; `expected_by_contract` = that exact value; assertion reads `defect_density` equals it. Deviation on any off value (wrong denominator, wrong scale, wrong rounding).
- label `density_excludes_test_files`, role `test_file_exclusion`, endpoint_role `defect_density_report_endpoint`, method POST, recipe `{ "kind": "test_file_exclusion", "fixture_role": "sprint_fixture_with_test_files" }` — fixture where test files carry a large line count; contract-correct denominator EXCLUDES them; `expected_by_contract` = density computed on non-test lines only. Deviation if the endpoint counts test-file lines (density too low) — surfaces a test-file-exclusion bug.
- label `density_rounding_half_up`, role `rounding_boundary`, endpoint_role `defect_density_report_endpoint`, method POST, recipe `{ "kind": "rounding_boundary", "fixture_role": "sprint_fixture_x_x5" }` — a fixture whose raw density ends in …x5 at the 3rd decimal so half-up vs half-even diverge; `expected_by_contract` = the half-up value. Deviation if the endpoint truncates or rounds half-even.

### GROUP B — severity weighting (P1=8, P2=4, P3=2, P4=1)
- label `severity_weighted_correct`, role `severity_weighting`, endpoint_role `defect_density_report_endpoint`, method POST, recipe `{ "kind": "severity_weighted_computation", "fixture_role": "sprint_fixture", "field_under_test": "severity_weighted_density" }` — `expected_by_contract` = Σ(weight×count) / non-test changed lines × 1000 half-up; assertion on `severity_weighted_density`. Deviation on wrong weights or wrong denominator.
- label `severity_weight_ordering`, role `severity_weight_swap`, endpoint_role `defect_density_report_endpoint`, method POST, recipe `{ "kind": "severity_weight_swap_probe", "fixture_role": "sprint_fixture_p1_heavy" }` — a P1-heavy fixture where a swapped/uniform weighting yields a materially different number; `expected_by_contract` uses the correct 8/4/2/1 order. Surfaces inverted or equal-weight severity bugs.

### GROUP C — P1–P4 tally
- label `p1_p4_counts_correct`, role `priority_tally`, endpoint_role `defect_density_report_endpoint`, method POST, recipe `{ "kind": "priority_tally", "fixture_role": "sprint_fixture" }` — `expected_by_contract` = the exact non-negative integer counts of P1/P2/P3/P4, each defect counted under exactly one key; assertion on `p1_count`..`p4_count`. Deviation on miscount, double-count, or unbucketed defect.
- label `priority_case_and_unknown`, role `priority_normalization`, endpoint_role `defect_density_report_endpoint`, method POST, recipe `{ "kind": "priority_normalization_probe", "fixture_role": "sprint_fixture_mixed_case_priority" }` — fixture with mixed-case/whitespace-padded priority labels; `expected_by_contract` = correct bucketing after normalization (or a 4xx validation error for an unknown/malformed priority — see Group G). Surfaces silent misbucketing.

### GROUP D — per-area rollup
- label `per_area_densities_correct`, role `per_area_rollup`, endpoint_role `defect_density_report_endpoint`, method POST, recipe `{ "kind": "per_area_rollup", "fixture_role": "sprint_fixture_with_component_map" }` — `expected_by_contract` = a map from each component label to its own defects / its own non-test changed lines × 1000 half-up; assertion compares each `per_area_densities[component]` entry. Deviation if a component's lines/defects are mis-attributed or the map omits a component.  (Emit only when a component-label map fixture is supplied.)
- label `per_area_no_cross_leak`, role `per_area_attribution`, endpoint_role `defect_density_report_endpoint`, method POST, recipe `{ "kind": "per_area_attribution_probe", "fixture_role": "sprint_fixture_shared_lines" }` — fixture where a file maps to one component only; `expected_by_contract` attributes its lines to that component alone. Surfaces cross-area attribution leakage.

### GROUP E — rolling average, deviation %, alert threshold, trend
- label `rolling_average_window`, role `rolling_window`, endpoint_role `defect_density_report_endpoint`, method POST, recipe `{ "kind": "rolling_average_window", "fixture_role": "sprint_fixture_with_3_prior" }` — `expected_by_contract` = mean of this sprint's density + the three preceding densities (a 4-value window) half-up; assertion on `rolling_3_sprint_average`. Deviation if the window includes the wrong count (2 or 4 priors), wrong ordering, or excludes the current sprint — off-window bug.
- label `deviation_percent_correct`, role `deviation_percent`, endpoint_role `defect_density_report_endpoint`, method POST, recipe `{ "kind": "deviation_percent_computation", "fixture_role": "sprint_fixture_with_3_prior" }` — `expected_by_contract` = (density − rolling_avg) / rolling_avg × 100 half-up; assertion on `deviation_percent`. Deviation on wrong sign, wrong base, or wrong scale.
- label `alert_fires_over_threshold`, role `alert_true`, endpoint_role `defect_density_report_endpoint`, method POST, recipe `{ "kind": "alert_threshold", "fixture_role": "sprint_fixture_deviation_gt_20" }` — |deviation| > 20 fixture; `expected_by_contract.alert` = true; assertion on `alert`. Surfaces an alert that fails to fire.
- label `alert_silent_at_boundary`, role `alert_boundary`, endpoint_role `defect_density_report_endpoint`, method POST, recipe `{ "kind": "alert_threshold_boundary", "fixture_role": "sprint_fixture_deviation_eq_20" }` — |deviation| exactly 20 (strict `>` means NOT firing); `expected_by_contract.alert` = false. Surfaces off-by-one / false-firing threshold bugs.
- label `trend_sign_correct`, role `trend_direction`, endpoint_role `defect_density_report_endpoint`, method POST, recipe `{ "kind": "trend_direction", "fixture_role": "sprint_fixture_density_lower_than_prior" }` — density below the immediately preceding sprint; `expected_by_contract.trend` = "down"; assertion on `trend`. Deviation if the endpoint reports "up"/"flat" — inverted-trend-sign bug.

### GROUP F — degenerate / malformed numeric input (must be safe, never 5xx)
- label `zero_changed_lines_no_div0`, role `division_by_zero`, endpoint_role `defect_density_report_endpoint`, method POST, recipe `{ "kind": "zero_lines_guard", "fixture_role": "sprint_fixture_zero_changed_lines" }` — all files excluded / zero changed lines; `expected_by_contract` = density 0.00 (guarded), status 2xx; assertion on `defect_density` == 0.00 AND no 5xx. Deviation on NaN/Infinity/500 — division-by-zero bug.
- label `negative_or_malformed_numstat`, role `malformed_input`, endpoint_role `defect_density_report_endpoint`, method POST, recipe `{ "kind": "malformed_numstat_validation", "fixture_role": "sprint_fixture_negative_lines" }` — negative/non-integer added-or-deleted counts; `expected_by_contract` = 4xx (400/422) machine-readable validation error, NEVER 5xx; assertion on status class 4xx. Surfaces unhandled bad input.
- label `empty_defect_set`, role `empty_sample`, endpoint_role `defect_density_report_endpoint`, method POST, recipe `{ "kind": "empty_defect_set", "fixture_role": "sprint_fixture_no_defects" }` — zero defects, non-zero lines; `expected_by_contract` = density 0.00, all p-counts 0, status 2xx. Deviation on error or non-zero output.

### GROUP G — standard API-contract cases on the reporting endpoint
- label `report_schema_conformance`, role `response_schema`, endpoint_role `defect_density_report_endpoint`, method POST, recipe `{ "kind": "report_schema_validation", "fixture_role": "sprint_fixture" }` — `expected_by_contract` = 200 with a body matching the documented report schema (all twelve fields present and typed, application/json); assertion validates the envelope. (Schema-shape only; deep JSON-schema strictness stays with `validate-json-schema-responses` — see de-dup.)
- label `report_requires_auth`, role `authn`, endpoint_role `defect_density_report_endpoint`, method POST, recipe `{ "kind": "unauthenticated_request", "fixture_role": "sprint_fixture" }` — missing/invalid credential; `expected_by_contract` = 401 (oracle AuthN row); assertion on status 401.
- label `report_missing_required_field_4xx`, role `validation`, endpoint_role `defect_density_report_endpoint`, method POST, recipe `{ "kind": "missing_required_fixture_field", "fixture_role": "sprint_fixture_missing_numstat" }` — fixture missing the numstat; `expected_by_contract` = 4xx (400/422), NEVER 5xx; assertion on status class.

New total: 20 cases. GROUP D depends on a component-label-map fixture; when absent, omit exactly those cases and fail the count check rather than fabricating a surface.

Closed recipe vocabulary AFTER this change (kinds): density_computation, test_file_exclusion, rounding_boundary, severity_weighted_computation, severity_weight_swap_probe, priority_tally, priority_normalization_probe, per_area_rollup, per_area_attribution_probe, rolling_average_window, deviation_percent_computation, alert_threshold, alert_threshold_boundary, trend_direction, zero_lines_guard, malformed_numstat_validation, empty_defect_set, report_schema_validation, unauthenticated_request, missing_required_fixture_field.

No regression-comparison case (that is `run-regression-suite`) and no satisfaction case (that is `measure-api-consumer-satisfaction`). All defect-density computation/reporting deviations are reserved to this owner in the map.

## ADDENDUM (v2 — exhaustive test-case + reporting standard)
When running update-agent for this agent, pass the Change prompt above AND this ADDENDUM together as a single change prompt.

This agent is a pure, exhaustive test-case generator in its lane. It fills the Expected Result (the contract-correct computation the defect-density reporting endpoint MUST return for a fixed fixture, sourced from the contract oracle and the researched formulas) and leaves `actual_result = "TO BE FILLED DURING EXECUTION"` and `status = Not Executed`. It emits NO deviations, NO verdict, NO pass/fail — a separate judge agent submits each fixture, records the endpoint's real output, and decides whether it is a bug. This ADDENDUM is governed by `00-AUTHORING-STANDARD-exhaustive-testcases.md`.

Test Case ID prefix: TC-DEFECT-NNN (zero-padded, sequential, stable).

Render every case in the human-readable schema with these fields: `test_case_id`, `title`, `description`, `category` (happy|negative|boundary|edge|broad), `feature_under_test` (the defect-density reporting endpoint that SERVES the metric), `preconditions`, `test_data`, `test_steps`, `expected_result`, `actual_result` (="TO BE FILLED DURING EXECUTION"), `status` (=`Not Executed`), `postconditions`, `severity_hint`, `references`, `tags`. Preserve this agent's existing machine fields (`label`, `role`, `endpoint_role`, `method`, `recipe`, `expected_by_contract`, `expected_by_docs`, `also_accept`, `assertion`) under a `machine` key inside each case. Emit ONE JSON object with a `test_cases[]` array.

### Lane-specific exhaustive coverage checklist (feature under test = the endpoint that computes/serves the defect-density metric; in-lane, MECE across agents)

- **Happy path (correct computation returned)**: core density = defects/(non-test changed lines)×1000 half-up returned exactly; severity-weighted density (P1=8/P2=4/P3=2/P4=1) correct; P1–P4 tally exact with each defect in exactly one bucket; rolling 3-sprint average over the correct 4-value window; deviation % correct; trend sign "down" when density falls; schema-conformant 200 with all twelve fields typed.
- **Negative path (malformed input → 4xx not 5xx)**: negative/non-integer numstat → 4xx machine-readable validation error, never 5xx; fixture missing the numstat → 4xx; missing/invalid credential → 401; unknown/malformed priority label → 4xx or documented bucketing.
- **Boundary (zero-sample / div-by-zero / threshold edges)**: zero changed lines → guarded density 0.00, status 2xx, no NaN/Infinity/500; empty defect set (non-zero lines) → density 0.00, all p-counts 0; alert exactly at |deviation|=20 → NOT firing (strict `>`); rounding fixture ending …x5 at the 3rd decimal → half-up value.
- **Edge (duplicate / outlier / rounding)**: mixed-case/whitespace-padded priority labels normalized without silent misbucketing; alert fires when |deviation|>20; test-file lines excluded from the denominator (density not deflated); per-area attribution has no cross-area leak (each file's lines to its one component).
- **Broad / combinatorial (each formula field × input shape)**: each computed field (defect_density, severity_weighted_density, per_area_densities, rolling_3_sprint_average, deviation_percent, alert, p1..p4, trend) × a fixture that isolates its correct value; per-area rollup over each component × its own non-test lines; each degenerate input (zero-lines, empty-defects, negative-numstat, missing-field) × safe-response class.

Adjacent concerns are cited to their owners: deep response-body JSON-schema strictness (additionalProperties, nested typing) → `validate-json-schema-responses` (here schema case proves only the twelve fields exist and are typed); error-message wording/leak → `verify-error-message-clarity` (here 4xx cases assert only status class + machine-readable error presence); credential lifecycle (expiry/revocation/JWT) → `test-authentication-flows`; regression-comparison → `run-regression-suite`; satisfaction metrics → `measure-api-consumer-satisfaction`.

Coverage is exhaustive in-lane and MECE across agents — no duplicate cases within this agent or with any sibling.
